# 測試指南（開發模式）

> 操作層文件：必跑自動 gate 與手動驗收步驟。與規格同期撰寫（acceptance-test 先行），供實作與 verifier 據以驗收。
> **角色歸屬**：verifier 維護與執行。實作者照清單驗收、不得放寬清單、不改本檔（只列建議）。
> 每項對應 `開發模式規格書.md` 的「### 驗收」段；同一條斷言不抄兩份，本檔只寫怎麼操作、看什麼。

## 測試環境

- 以專案 venv 啟動：`start_devmode.cmd`（內部沿用 `start.cmd` 的 venv 檢查）。
- **玩具專案**：另建一個拋棄式目錄（如 `C:\_work\tmp\devmode-toy`），`git init` 並 commit 一個小規格檔（例如 `spec.md`：「寫一個 `add.py`，含 `add(a,b)` 函式與一個測試」）。所有可寫測試都對玩具專案跑，不要拿真專案測。
- 真 CLI 呼叫花錢花時間：全管線集中在一次「玩具任務全流程」；協議錯誤、限流、仲裁、續作與訊息路由由 `tests/test_devmode.py` 的 fake adapter／暫存 repo 確定性覆蓋，不臨時修改正式程式。

## Phase D1 — 模式進入與權限（規格 §3、§4）

1. **互斥**：PowerShell 設 `$env:AI_ROUNDTABLE_PUBLIC='1'; $env:AI_ROUNDTABLE_DEVMODE='1'` 後啟動 → 伺服器直接退出，訊息含互斥說明。
2. **只綁 loopback**：devmode 啟動後 `Get-NetTCPConnection -LocalPort 8787 -State Listen` → 只有 `127.0.0.1`，無 Tailscale IP。
3. **邀請停用**：頁面無「邀請」鈕；`POST /api/token/generate` 回 403。
4. **管線啟動前的討論行為不變**：devmode 下正常發言 → 已啟用席位照常回覆；勾共識討論模式跑一輪 → 行為同討論版；呼叫中按「取消」→ 單席位取消成功。
5. **角色由使用者自控**：把 `dev_roles` 改成合法且互異的非預設映射後重啟 → `/api/state.dev.roles` 原樣回傳，重新載入 settings 後仍保留；不必用非預設映射真的跑可寫管線，測完恢復預設。`roles_custom` 與畫面警告歸 D3 驗收。
6. **討論版不受影響**：改用 `start.cmd` 啟動 → 無徽章、無 ▶ 鈕，agy 參數仍是 `--sandbox`；非法或重複的 `dev_roles` 回落預設。

## Phase D2 — 管線核心與安全網（規格 §5、§6、§7）

### 必跑自動 gate

7. 執行 `.\.venv\Scripts\python.exe -m unittest tests.test_devmode -v`，必須全部 PASS，且不得啟動真模型 CLI：
   - [ ] 合法任務協議可解析；缺 `files`、錯型別、空 acceptance、重複／非正整數 id、空白驗證原因與仲裁內容全被拒絕；digest 空清單保留 done 並進入 handoff，不重新 dispatch。
   - [ ] 協議首次失敗會重問一次，連續失敗轉 `parse_fail`；stderr 含限流標記時 attempts 不變。
   - [ ] turn cap 續作把計數歸零；管線分支髒 tree 先產生「中斷殘留變更」commit，非管線分支髒 tree 仍拒絕。
   - [ ] 仲裁「詢問」不設 arbitrated；主持人訊息後續作先跑消化，任務不會直接 auto-block。
   - [ ] tasks.json 被竄改時，管線以最後 trusted snapshot 暫停，不得把注入任務重新保存成可信狀態。

### 前置檢查

8. 玩具專案先 `echo x > dirty.txt`（不 commit）→ 按「▶」→ 拒絕，逐字稿說明髒 working tree；還原後重按 → 通過。
9. 對非 git 目錄啟動 devmode → 按「▶」→ 拒絕，訊息建議 `git init`。

### 玩具任務全流程（一次跑完，逐項核對）

10. 開場發言交代玩具目標 → 按「▶」：
   - [ ] 主控拆任務訊息出現，`data/tasks.json` 生成且內容合法（每任務含 id/title/files/acceptance）。
   - [ ] `git branch` 顯示新的 `roundtable/dev-*` 分支且為當前分支。
   - [ ] 實作棒後分支上出現 `[roundtable] 任務1 ...` commit；main 無新 commit。
   - [ ] 驗證棒訊息最後一行是「驗證結果：…」；通過後 tasks.json 該任務 `status: done`。
   - [ ] 全部完成後出現主控收尾摘要，`tasks.json.status == "done"`。
   - [ ] `data/devlogs/` 每棒一檔，內含 instruction 與完整 stdout/stderr。
11. **失敗迴圈**：玩具規格故意寫一條無法滿足的驗收（如「測試必須涵蓋除以零回傳 42」）→ 觀察：不通過意見出現在下一實作棒日誌的 instruction 內；第 3 次失敗後出現仲裁棒，最後一行為「仲裁：…」。

### 安全與續作分支

12. **預設席位參數**：跑一棒管線後查看 `data/devlogs/` args——agy 含 `--dangerously-skip-permissions`、codex 含 `--sandbox read-only`、claude 的 `--allowedTools` 僅 `Read,Glob,Grep`。
13. **竄改**：管線暫停時手動編輯 `data/roundtable.md` 任一字 → 按「▶」→ 下一棒開始前停線、逐字稿出現竄改警告、devBar 顯示警告狀態。
14. **crash 續作**：管線執行中直接關掉 console → 重啟 devmode → `tasks.json.status == "paused"`、`pause_reason: crash` → 按「▶」→ 從中斷任務續跑。
15. **中斷殘留 commit**：在已暫停的管線分支手動建立一個未提交檔案 → 按「▶」→ 先出現 `[roundtable] 任務{id} 中斷殘留變更` commit，該 hash 記入目前任務後才續跑；相同行為在非管線分支必須被拒。
16. **總棒數上限與重新授權**：以 `AI_ROUNDTABLE_DEV_MAX_TURNS=2` 跑到 `turn_cap` → 按「▶」→ `turn_count` 先歸零且至少能再跑一棒，不得立刻再次因舊計數暫停。
17. **原子性**（code review 項）：確認所有 tasks.json 寫入走 `_write_atomic`；grep 不得出現直接 `TASKS_FILE.write_text`。

## Phase D3 — UI 與續作體驗（規格 §8）

18. 徽章：devmode 頁首有「開發模式」標示；`start.cmd` 版沒有。
19. 按鈕：閒置=「▶ 開始執行」；執行中=「⏸ 暫停」；guest 不適用（devmode 無 guest）；非 HOST 情境不出現。
20. devBar：拆任務／實作／驗證各階段顯示「任務 m/n · 階段 · 分支名」；暫停顯示原因；顯示角色映射，非預設時顯示能力警告；竄改顯示警告。
21. **執行中插話**：先以 unittest 將 `start_batch`／`start_discussion` 換成會立即失敗的 stub；送出「任務 2 改用 pathlib」→ 訊息只進逐字稿、stub 均未被呼叫，也沒有一般 AI 回覆或額外 subprocess；當前棒跑完後出現主控消化棒，done 任務不可變，之後自動續跑。
22. **暫停期間發言**：按 ⏸ 並等當前棒結束 → 發言後不出現一般 AI 回覆且不自動續跑 → 再按 ▶，第一棒必須是主控消化；仲裁「詢問」的主持人拍板也走同一路徑且不得直接 blocked。
23. 新會議：管線 active／paused 時按「🧹 新會議」→ 被拒並提示原因；管線 done 後可開新會議，封存目錄出現對應 `tasks-*.json`。

## 測試原則

- `tests/test_devmode.py` 是 v1 必跑 gate，不是建議項；安全狀態分支不得依賴臨時改 prompt、低階模型碰運氣或正式程式隱藏鉤子。
- 真 CLI 玩具任務只驗證 adapter 與端到端整合；不能取代 fake adapter 對所有狀態轉換的確定性覆蓋。
