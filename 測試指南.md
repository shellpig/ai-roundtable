# 測試指南（開發模式）

> 操作層文件：手動驗收步驟清單。與規格同期撰寫（acceptance-test 先行），供實作時據以自我驗收。
> **角色歸屬**：verifier 維護與執行。實作者照清單驗收、不得放寬清單、不改本檔（只列建議）。
> 每項對應 `開發模式規格書.md` 的「### 驗收」段；同一條斷言不抄兩份，本檔只寫怎麼操作、看什麼。

## 測試環境

- 以專案 venv 啟動：`start_devmode.cmd`（內部沿用 `start.cmd` 的 venv 檢查）。
- **玩具專案**：另建一個拋棄式目錄（如 `C:\_work\tmp\devmode-toy`），`git init` 並 commit 一個小規格檔（例如 `spec.md`：「寫一個 `add.py`，含 `add(a,b)` 函式與一個測試」）。所有可寫測試都對玩具專案跑，不要拿真專案測。
- 真 CLI 呼叫花錢花時間：跑全管線的項目集中在一次「玩具任務全流程」完成多項驗收；純狀態機分支（解析失敗、限流、竄改）用下方的人工誘發法，不必真跑多輪。

## Phase D1 — 模式進入與權限（規格 §3、§4）

1. **互斥**：PowerShell 設 `$env:AI_ROUNDTABLE_PUBLIC='1'; $env:AI_ROUNDTABLE_DEVMODE='1'` 後啟動 → 伺服器直接退出，訊息含互斥說明。
2. **只綁 loopback**：devmode 啟動後 `Get-NetTCPConnection -LocalPort 8787 -State Listen` → 只有 `127.0.0.1`，無 Tailscale IP。
3. **邀請停用**：頁面無「邀請」鈕；`POST /api/token/generate` 回 403。
4. **討論行為不變**：devmode 下正常發言 → 已啟用席位照常回覆；勾共識討論模式跑一輪 → 行為同討論版；呼叫中按「取消」→ 單席位取消成功。
5. **席位參數**：跑一棒管線後開 `data/devlogs/` 對應日誌，確認 args——agy 含 `--dangerously-skip-permissions`、codex 含 `--sandbox read-only`、claude 的 `--allowedTools` 僅 `Read,Glob,Grep`。
6. **討論版不受影響**：改用 `start.cmd` 啟動 → 無徽章、無 ▶ 鈕，agy 參數仍是 `--sandbox`（可由程式碼或臨時日誌確認）。

## Phase D2 — 管線核心與安全網（規格 §5、§6、§7）

### 前置檢查

7. 玩具專案先 `echo x > dirty.txt`（不 commit）→ 按「▶」→ 拒絕，逐字稿說明髒 working tree；還原後重按 → 通過。
8. 對非 git 目錄啟動 devmode → 按「▶」→ 拒絕，訊息建議 `git init`。

### 玩具任務全流程（一次跑完，逐項核對）

9. 開場發言交代玩具目標 → 按「▶」：
   - [ ] 主控拆任務訊息出現，`data/tasks.json` 生成且內容合法（每任務含 id/title/acceptance）。
   - [ ] `git branch` 顯示新的 `roundtable/dev-*` 分支且為當前分支。
   - [ ] 實作棒後分支上出現 `[roundtable] 任務1 ...` commit；main 無新 commit。
   - [ ] 驗證棒訊息最後一行是「驗證結果：…」；通過後 tasks.json 該任務 `status: done`。
   - [ ] 全部完成後出現主控收尾摘要，`tasks.json.status == "done"`。
   - [ ] `data/devlogs/` 每棒一檔，內含 instruction 與完整 stdout/stderr。
10. **失敗迴圈**：玩具規格故意寫一條無法滿足的驗收（如「測試必須涵蓋除以零回傳 42」）→ 觀察：不通過意見出現在下一實作棒日誌的 instruction 內；第 3 次失敗後出現仲裁棒，最後一行為「仲裁：…」。

### 人工誘發分支（不必真跑多輪）

11. **解析失敗**：暫時把主控模型改成低階（或臨時在指令模板加「不要輸出 json」）誘發 → 重問一次 → 仍失敗 → 管線暫停、`pause_reason: parse_fail`。（建議：實作者可提供一個隱藏測試鉤子常數，verifier 用它注入假輸出；否則以臨時改模板方式測。）
12. **限流**：以測試鉤子讓某棒 stderr 含 `rate limit` → 管線暫停、該任務 `attempts` 不變、`pause_reason: rate_limit`。
13. **竄改**：管線暫停時手動編輯 `data/roundtable.md` 任一字 → 按「▶」→ 下一棒開始前停線、逐字稿出現竄改警告、devBar 顯示警告狀態。
14. **crash 續作**：管線執行中直接關掉 console → 重啟 devmode → `tasks.json.status == "paused"`、`pause_reason: crash` → 按「▶」→ 從中斷任務續跑。
15. **總棒數上限**：`$env:AI_ROUNDTABLE_DEV_MAX_TURNS='2'` 啟動 → 跑玩具任務 → 第 2 棒後自動暫停、`pause_reason: turn_cap`。
16. **原子性**（code review 項）：確認所有 tasks.json 寫入走 `_write_atomic`；grep 不得出現直接 `TASKS_FILE.write_text`。

## Phase D3 — UI 與續作體驗（規格 §8）

17. 徽章：devmode 頁首有「開發模式」標示；`start.cmd` 版沒有。
18. 按鈕：閒置=「▶ 開始執行」；執行中=「⏸ 暫停」；guest 不適用（devmode 無 guest）；非 HOST 情境不出現。
19. devBar：拆任務／實作／驗證各階段顯示「任務 m/n · 階段 · 分支名」；暫停顯示原因；竄改顯示警告。
20. 插話：管線執行中送出一則發言（如「任務 2 改用 pathlib」）→ 當前棒跑完後出現主控消化棒 → 任務清單更新（done 任務不可變）→ 自動續跑。⏸ 按下後則停住不續跑，再按 ▶ 才續。
21. 新會議：管線 active／paused 時按「🧹 新會議」→ 被拒並提示原因；管線 done 後可開新會議，封存目錄出現對應 `tasks-*.json`。

## 建議（非清單項）

- 若後續要做管線狀態機的自動化測試，方向是 mock adapter（假席位函式回預錄輸出）跑 §5/§6 全分支，不叫真 CLI；v1 未要求。
